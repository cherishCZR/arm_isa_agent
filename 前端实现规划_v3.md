# ARM ISA Compiler Verification Agent — 前端实现规划 v3

> **核心变更**：实时进度展示作为整个验证流程的一等公民，贯穿单指令验证、批量验证、对话模式三种场景。后端通过 SSE 流式推送各阶段状态，前端以实时过程面板展示 Pipeline 执行进度。

---

## 1. 实时进度设计概述

### 1.1 用户看到的实时进度

用户提交验证请求后，前端不再只显示 loading spinner 然后一次性返回结果，而是实时展示：

```
┌──────────────────────────────────────────────────────────────────┐
│  正在验证: ADD                                                    │
│                                                                   │
│  ● Planner          ✅ 完成 (8ms)                                  │
│    Analyzing ADD constraints...                                   │
│                                                                   │
│  ● Retrieval        ✅ 完成 (12ms)                                 │
│    Found ARM ARM XML entry...                                     │
│                                                                   │
│  ● Generator        ⏳ 运行中 (180ms)...                            │
│    Creating boundary cases...                                     │
│                                                                   │
│  ○ Reviewer         等待中                                        │
│                                                                   │
│  ○ Repair           等待中                                        │
│                                                                   │
│  ──────────────────────────────────────────────────────────────── │
│  总耗时: 200ms                                                    │
└──────────────────────────────────────────────────────────────────┘
```

各阶段依次点亮：
- **Planner** → "Analyzing ADD constraints..."
- **Retrieval** → "Found ARM ARM XML entry..."  
- **Generator** → "Creating boundary cases..."
- **Reviewer** → "Detected immediate overflow..."
- **Repair** → "Fixed testcase..."

每个阶段完成后展示该阶段的 **耗时** 和 **关键数据快照**（如编码变体数、约束条数、生成测试数）。

### 1.2 三种场景都要有实时进度

| 场景 | 进度展示方式 |
|------|-------------|
| **单指令验证** `/verify` | 提交后全屏跑马灯式 Pipeline 进度面板（5 个阶段依次点亮），完成后再展示推理时间线 + 测试文件表格 |
| **批量验证** `/batch` | 左侧指令列表（每行实时更新状态 + 当前阶段），右侧当前正在处理的指令的 Pipeline 进度面板 |
| **对话模式** `/chat` | 对话气泡内嵌入小型进度条（5 个阶段点），完成后切换为推理摘要折叠 + 代码块 |

### 2.3 Orchestrator SSE 改造

`VerificationOrchestrator` 新增 `verify_stream()` 异步生成器方法：

```python
# verification/orchestrator.py — 新增

from typing import AsyncGenerator
import json

class StageEvent:
    """单个 SSE 事件。"""
    event: str          # stage_start | stage_progress | stage_complete | result | done
    data: dict[str, Any]

class VerificationOrchestrator:
    # ... 现有代码保持不变 ...

    async def verify_stream(self, instruction: str) -> AsyncGenerator[str, None]:
        """流式验证，yield SSE 格式字符串。"""
        yield self._sse("stage_start", {
            "stage": "planner",
            "message": f"Analyzing {instruction} constraints...",
        })

        # Stage 1: 约束分析 + 测试规划 (合并为 Planner)
        profile = self.analyzer.extract_profile(mnemonic=instruction)
        yield self._sse("stage_complete", {
            "stage": "planner",
            "status": "ok",
            "duration_ms": ...,
            "summary": f"Found {n} constraints: ...",
            "findings": [...],
            "snapshot": {...},
        })

        yield self._sse("stage_start", {
            "stage": "retrieval",
            "message": f"Retrieving instruction profile from knowledge base...",
        })
        # ... 各阶段依次 yield ...

        yield self._sse("result", {
            "instruction": instruction,
            "status": overall_status,
            "review_score": review_score,
            "test_files": [...],
            ...
        })
        yield self._sse("done", {})

    @staticmethod
    def _sse(event: str, data: dict) -> str:
        return f"event: {event}\ndata: {json.dumps(data)}\n\n"
```

### 2.4 Stage 名称映射

后端内部 6 阶段 → 前端展示 5 阶段：

| 后端内部名称 | SSE stage 名称 | 前端展示 | 典型 message |
|-------------|---------------|---------|-------------|
| `constraint_analysis` + `test_planning` | `planner` | Planner | "Analyzing ADD constraints..." |
| `retrieval` | `retrieval` | Retrieval | "Found ARM ARM XML entry..." |
| `test_generation` | `generator` | Generator | "Creating boundary cases..." |
| `test_review` | `reviewer` | Reviewer | "Detected immediate overflow..." |
| `repair` | `repair` | Repair | "Fixed testcase..." |

> 注：后端 orchestrator 内部仍然保持 6 阶段不变，`verify_stream()` 中将 `constraint_analysis` + `test_planning` 合并为一个 `planner` stage 向外暴露。

### 4.4 TypeScript 类型定义

```typescript
// lib/api/types.ts

// ── SSE 事件类型 ──

type SSEEventName =
  | "stage_start"
  | "stage_progress"
  | "stage_complete"
  | "instruction_start"
  | "instruction_complete"
  | "batch_start"
  | "batch_complete"
  | "result"
  | "done";

type PipelineStage = "planner" | "retrieval" | "generator" | "reviewer" | "repair";
type StageStatus = "pending" | "running" | "ok" | "warning" | "error" | "skipped";

interface StageStartEvent {
  stage: PipelineStage;
  message: string;               // "Analyzing ADD constraints..."
  timestamp: number;
  instruction?: string;          // batch 模式下携带
}

interface StageProgressEvent {
  stage: PipelineStage;
  detail: string;                // "正在生成 Normal 测试 (assembly)... 4/4"
  instruction?: string;
}

interface StageCompleteEvent {
  stage: PipelineStage;
  status: StageStatus;
  duration_ms: number;
  summary: string;
  findings: string[];
  snapshot: Record<string, unknown>;
  instruction?: string;
}

// ── 数据模型 ──

interface TestFileEntry {
  file_id: string;
  filename: string;
  format: "s" | "c" | "cpp" | "llvm";
  test_type: string;
  content: string;
  status: "pass" | "fail";
  description: string;
  issue_count: number;
}

interface StageDetail {
  stage: PipelineStage;
  status: StageStatus;
  duration_ms: number;
  summary: string;
  reasoning: string;             // 完整推理链
  key_findings: string[];
  data_snapshot: Record<string, unknown>;
}

interface VerificationResult {
  instruction: string;
  status: "PASS" | "FAIL" | "REPAIRED" | "ERROR";
  review_score: number;
  total_tests: number;
  passing_tests: number;
  failing_tests: number;
  total_duration_ms: number;
  stage_details: StageDetail[];
  test_files: TestFileEntry[];
  coverage: Record<string, number>;
  issues: Array<Record<string, unknown>>;
}

// ── 前端运行时状态 ──

interface PipelineState {
  stages: Record<PipelineStage, {
    status: StageStatus;
    message: string;
    duration_ms: number;
    findings: string[];
  }>;
  current_stage: PipelineStage | null;
  is_running: boolean;
  total_duration_ms: number;
}
```

**交互规则：**
- 每个阶段依次出现在时间线上
- 当前运行阶段显示 ⏳ 动画 + 动态子消息滚动（如 Generator 内部生成各类测试的进度）
- 已完成阶段可 **点击展开** 查看详细发现（编码变体数、约束条数等）
- 用户可随时点击 [取消验证] 中断 SSE 连接

### 6.1 SSE 消费 Hook (POST SSE)

```typescript
// lib/hooks/use-sse.ts

export function useVerificationSSE() {
  const [pipeline, setPipeline] = useState<PipelineState>(initialPipelineState());
  const [result, setResult] = useState<VerificationResult | null>(null);
  const [isRunning, setIsRunning] = useState(false);
  const abortRef = useRef<AbortController | null>(null);

  const startVerification = async (instruction: string) => {
    const controller = new AbortController();
    abortRef.current = controller;
    setIsRunning(true);
    resetPipeline();

    const response = await fetch("/api/generate_testcase/stream", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ instruction }),
      signal: controller.signal,
    });

    const reader = response.body!.getReader();
    const decoder = new TextDecoder();
    let buffer = "";

    while (true) {
      const { done, value } = await reader.read();
      if (done) break;

      buffer += decoder.decode(value, { stream: true });
      const lines = buffer.split("\n");
      buffer = lines.pop() || "";

      let eventName = "";
      for (const line of lines) {
        if (line.startsWith("event: ")) {
          eventName = line.slice(7);
        } else if (line.startsWith("data: ") && eventName) {
          const data = JSON.parse(line.slice(6));
          handleEvent(eventName, data);
        }
      }
    }

    setIsRunning(false);
  };

  const handleEvent = (event: SSEEventName, data: any) => {
    switch (event) {
      case "stage_start":
        setPipeline(prev => updateStage(prev, data.stage, {
          status: "running",
          message: data.message,
        }));
        break;
      case "stage_progress":
        setPipeline(prev => updateStage(prev, data.stage, {
          message: data.detail,  // 动态更新子消息
        }));
        break;
      case "stage_complete":
        setPipeline(prev => updateStage(prev, data.stage, {
          status: data.status,
          duration_ms: data.duration_ms,
          summary: data.summary,
          findings: data.findings,
        }));
        break;
      case "result":
        setResult(data);
        break;
      case "done":
        setIsRunning(false);
        break;
    }
  };

  const cancel = () => abortRef.current?.abort();

  return { pipeline, result, isRunning, startVerification, cancel };
}
```

import JSZip from "jszip";
import { saveAs } from "file-saver";

export function useDownload() {
  const downloadSingle = (entry: TestFileEntry) => {
    const blob = new Blob([entry.content], { type: "text/plain" });
    saveAs(blob, entry.filename);
  };

  const downloadAllAsZip = async (
    instruction: string,
    entries: TestFileEntry[],
    onlyPassing = true
  ) => {
    const zip = new JSZip();
    const folder = zip.folder(instruction)!;
    const files = onlyPassing
      ? entries.filter(e => e.status === "pass")
      : entries;
    files.forEach(e => folder.file(e.filename, e.content));
    const blob = await zip.generateAsync({ type: "blob" });
    saveAs(blob, `${instruction}_testcases.zip`);
  };

  const batchDownloadAsZip = async (
    results: VerificationResult[]
  ) => {
    const zip = new JSZip();
    results.forEach(r => {
      const folder = zip.folder(r.instruction)!;
      r.test_files
        .filter(f => f.status === "pass")
        .forEach(f => folder.file(f.filename, f.content));
    });
    const blob = await zip.generateAsync({ type: "blob" });
    const date = new Date().toISOString().slice(0, 10);
    saveAs(blob, `arm_testcases_${date}.zip`);
  };

  return { downloadSingle, downloadAllAsZip, batchDownloadAsZip };
}

| 场景 | 默认展示 | 下载按钮 | 文件格式 |
|------|----------|----------|----------|
| "帮我生成 ADD 的测试" | 代码块 + 📋复制 | 每文件下方 💾.s 💾.c 💾.cpp | 自动检测 |

4. 所有场景 → 实时进度面板作为首要 UX
