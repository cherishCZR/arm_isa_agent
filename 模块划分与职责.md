# 模块划分与职责

## 模块全景

| 模块 | 职责 | 类型 |
|------|------|------|
| arm_isa_agent/core | 核心基础设施：配置、领域模型、异常、常量 | 基础设施 |
| arm_isa_agent/etl | XML 解析 → SQLite + ChromaDB 数据写入 | 数据管道 |
| arm_isa_agent/kb | 双知识库抽象层，统一查询接口 (SQLite + ChromaDB) | 数据访问 |
| arm_isa_agent/agent | LangGraph Agent 运行时 (Router/Planner/Executor/Summarizer) | 编排层 |
| arm_isa_agent/tools | 工具适配层：将领域模块暴露为 Agent 可调用的工具 | 适配层 |
| arm_isa_agent/rag | RAG 检索领域逻辑 (BM25 + Embedding + 混合检索) | 领域逻辑 |
| arm_isa_agent/planning | 测试计划领域逻辑 (TestPlan / Profile 分析) | 领域逻辑 |
| arm_isa_agent/generation | 测试生成领域逻辑 (规则 + LLM 辅助) | 领域逻辑 |
| arm_isa_agent/review_generation | Review + Self-Correction 领域逻辑 (5 维评分) | 领域逻辑 |
| arm_isa_agent/verification | Compiler Verification 端到端流水线编排 | 领域逻辑 |
| arm_isa_agent/api | FastAPI 对外接口 (Verification API) | 接口层 |
| arm_isa_agent/cli | 命令行管理工具 | 接口层 |

## 架构分界：编排层 vs 领域层 vs 适配层

```
┌──────────────────────────────────────────────────────┐
│  agent/  (LangGraph 运行时)                           │  ← 编排层
│  graph.py / state.py / tool_registry.py / prompt_manager.py
└────────────────────────┬─────────────────────────────┘
                         │ 通过 @register_tool 解耦
┌────────────────────────▼─────────────────────────────┐
│  tools/  (适配层)                                    │  ← 适配层
│  retrieval.py / planning.py / testgen.py / ...       │
│  领域函数 → Agent可调用工具                            │
└────┬──────────┬──────────┬──────────┬────────────────┘
     │          │          │          │
┌────▼───┐ ┌───▼────┐ ┌───▼────┐ ┌───▼───────────┐
│  rag/  │ │planning│ │generati│ │review_generati│   ← 领域层
│        │ │ /      │ │ on/    │ │ on/           │
└────────┘ └────────┘ └────────┘ └───────────────┘
     │          │          │          │
     └──────────┴──────────┴──────────┘
                       │
              ┌────────▼────────┐
              │  verification/  │  ← 端到端流水线 (调用所有领域模块)
              └─────────────────┘
```

**设计原则**:
- `agent/` 只做编排，不持有领域细节 → 新增领域模块无需修改 agent 核心
- `tools/` 是适配层，把领域函数包装为 Agent 可调用的工具 (带 schema)
- `rag/` / `planning/` / `generation/` / `review_generation/` / `verification/` 是独立领域模块，可独立测试
- `verification/` 是顶层编排器，串联所有领域模块形成完整流水线

## 详细模块职责

### 1. arm_isa_agent/core — 核心基础设施

| 子模块 | 职责 |
|--------|------|
| config.py | 全局配置 (pydantic-settings: DB 路径、LLM API Key、Embedding 模型等) |
| constants.py | 枚举常量 (InstructionClass, OperandType, ConstraintType, BitWidth 等) |
| exceptions.py | 统一异常层级 (KBError, ParseError, AgentError, APITimeout...) |
| models/instruction.py | Pydantic 领域模型 (Instruction, Encoding, Operand, Feature 等) |

### 2. arm_isa_agent/etl — ETL 管道

| 子模块 | 职责 |
|--------|------|
| xml_parser.py | 递归遍历 XML 目录，lxml 流式解析 instructionsection → ParsedInstruction |
| constraint_extractor.py | 提取 CONSTRAINED UNPREDICTABLE 和 encoding-level constraints |
| instruction_card_builder.py | 将结构化数据转换为 Markdown 格式 Instruction Card（存入 ChromaDB） |
| pipeline.py | ETL 管道编排器：XML Parse → Extract → SQLite Write → ChromaDB Write |

### 3. arm_isa_agent/kb — 双知识库

| 子模块 | 职责 |
|--------|------|
| sqlite/client.py | SQLAlchemy 引擎 & Session 管理 |
| sqlite/models.py | ORM 模型：instructions, encodings, operands, features, constraints, pseudocode + M2M 关系表 |
| chroma/client.py | ChromaDB 连接管理、Collection 创建与查询 |

### 4. arm_isa_agent/agent — LangGraph Agent 运行时 (编排层)

| 子模块 | 职责 |
|--------|------|
| graph.py | StateGraph 定义与编译：Router → Planner → Executor → [Reviewer ⇄ Repair] → Summarizer |
| state.py | AgentState TypedDict：messages, user_query, intent, plan, tool_results, review_results, verification_report 等 |
| tool_registry.py | 工具注册中心：@register_tool 装饰器自动注册，生成 OpenAI function schema |
| prompt_manager.py | Prompt 模板管理 (System Prompt + Intent 路由 Prompt + Few-shot) |

**支持的 Intent 路由**:
| Intent | 流程 |
|--------|------|
| retrieval | Planner → Executor → Summarizer |
| agentic_search | Planner → Executor → Summarizer |
| general | Router → Summarizer (直出) |
| test_generation | Planner → Executor → Reviewer ⇄ Repair → Summarizer |
| verification | Planner → Executor → Reviewer ⇄ Repair → Summarizer |

### 5. arm_isa_agent/tools — 工具适配层

| 子模块 | 绑定领域 | 主要工具 |
|--------|----------|----------|
| retrieval.py | rag/ + kb/ | 语义搜索指令、获取编码、操作数、伪代码、约束 |
| planning.py | planning/ | generate_test_plan, generate_verification_plan |
| testgen.py | generation/ | generate_testcase_from_plan |
| generation.py | generation/ | generate_code 系列工具 |
| review.py | review_generation/ | review_code 系列工具 |
| review_gen.py | review_generation/ | review_testcase, repair_testcase |

### 6. arm_isa_agent/rag — RAG 检索领域逻辑 (Phase 4)

| 子模块 | 职责 |
|--------|------|
| pipeline.py | 检索管道编排：混合检索策略 (BM25 + Dense)，结果融合与重排序 |
| retriever.py | 检索器抽象接口 + 具体实现 (语义检索、结构化查询) |
| bm25.py | BM25 稀疏检索实现 (关键词匹配，快速召回) |
| embedding.py | Embedding 密集检索实现 (语义相似度，精确召回) |

### 7. arm_isa_agent/planning — 测试计划领域逻辑 (Phase 5)

| 子模块 | 职责 |
|--------|------|
| models.py | TestPlan, TestDimension, Strategy, ProfileAnalysis 数据模型 |
| prompts.py | 规划 Prompt 模板 (System / Few-shot) |
| planner.py | TestPlanner：根据指令 Profile 生成测试方案 (维度划分 + 优先级) |
| analyzer.py | ProfileAnalyzer：分析指令特性 (immediates / features / constraints) |

### 8. arm_isa_agent/generation — 测试生成领域逻辑 (Phase 6)

| 子模块 | 职责 |
|--------|------|
| models.py | TestCaseSuite, TestCase, ASMCall, DataSequence, DataElement 数据模型 |
| prompts.py | 生成 Prompt 模板 (规则生成 / LLM 辅助生成) |
| generators.py | 规则生成器 (8 种格式) + LLM 辅助生成器 |

**支持的测试格式**: ASM_EMPTY, ASM_COMMENT, ASM_PLAIN, ASM_OPERS, ASM_IMM0, ASM_SPIMM, ASM_FP_IMM, ASM_RNDR

### 9. arm_isa_agent/review_generation — Review + Self-Correction (Phase 7)

| 子模块 | 职责 |
|--------|------|
| models.py | ReviewResult (5 维评分), ReviewIssue, RepairResult 数据模型 |
| prompts.py | Review / Repair Prompt 模板 |
| reviewer.py | TestcaseReviewer (语法/语义/格式/边界/一致性 5 维) + RepairGenerator |

**Review 维度**:
| 维度 | 评分依据 |
|------|----------|
| syntax | 汇编语法正确性 |
| semantics | 语义与 ISA 规范一致性 |
| format | 输出格式符合测试框架要求 |
| boundary | 边界条件覆盖 |
| consistency | 与其他测试用例一致性 |

### 10. arm_isa_agent/verification — Compiler Verification 流水线 (Phase 8)

| 子模块 | 职责 |
|--------|------|
| models.py | VerificationReport, CoverageBreakdown, StageResult |
| orchestrator.py | VerificationOrchestrator：7 阶段端到端流水线编排 |

**7 阶段流水线**:
```
Stage 1: Instruction Retrieval   →  从 KB 获取指令完整 Profile
Stage 2: Constraint Analysis     →  分析 immediate / feature / UNPREDICTABLE 约束
Stage 3: Test Planning           →  生成覆盖维度 + 优先级策略
Stage 4: Test Generation         →  8 种格式，规则 + LLM 辅助生成
Stage 5: Test Review             →  5 维评分，输出 ReviewReport
Stage 6: Repair Loop (max 3)     →  自动修复检测到的问题
Stage 7: Verification Report     →  汇总 report (JSON/Markdown)，含 coverage + score
```

**VerificationReport 结构**:
```json
{
  "instruction": "ADD",
  "status": "passed",
  "generated_tests": 20,
  "coverage": {
    "boundary": 100,
    "alias": 80,
    "invalid": 100
  },
  "review_score": 96,
  "issues": []
}
```

### 11. arm_isa_agent/api — FastAPI 接口层

| 路由 | 方法 | 描述 |
|------|------|------|
| /api/generate_testcase | POST | 单指令 Compiler Verification (输入 instruction 名) |
| /api/generate_testcases | POST | 批量指令 Compiler Verification |
| /api/health | GET | 健康检查 |

### 12. arm_isa_agent/cli — CLI 管理工具

| 命令 | 描述 |
|------|------|
| arm-isa etl run | 运行完整 ETL 管道 |
| arm-isa kb stats | 知识库统计信息 |
| arm-isa serve | 启动 FastAPI 服务 (uvicorn) |
| arm-isa agent chat | 命令行交互式 Agent 对话 |

## 关键数据流

### Compiler Verification 端到端流程

```
User Request (instruction name)
        │
        ▼
┌──────────────────────────────┐
│ api/routes/verification.py   │  ← FastAPI endpoint
│ POST /api/generate_testcase  │
└────────────┬─────────────────┘
             ▼
┌──────────────────────────────┐
│ verification/orchestrator.py │  ← VerificationOrchestrator
│ run_pipeline(instruction)    │
└────────────┬─────────────────┘
             │
    ┌────────┼────────┬──────────┬──────────┐
    ▼        ▼        ▼          ▼          ▼
  Stage1   Stage2   Stage3     Stage4     Stage5-7
  (rag)  (planning) (planning) (generation) (review_gen)
    │        │        │          │          │
    └────────┴────────┴──────────┴──────────┘
             │
             ▼
    VerificationReport → JSON Response
```

### Agent 对话流程

```
User Query
    │
    ▼
┌──────────────────────────┐
│ agent/graph.py            │  ← LangGraph StateGraph
│ Router → intent 分类     │
└──────────┬───────────────┘
           ▼
┌──────────────────────────┐
│ Planner                  │  ← LLM 生成 tool-call plan
└──────────┬───────────────┘
           ▼
┌──────────────────────────┐
│ Executor (tool-call loop)│  ← 通过 tool_registry 调用 tools/
└──────────┬───────────────┘
           ▼
    ┌──────┴──────┐
    ▼             ▼
┌────────┐  ┌──────────────┐
│Summary │  │Reviewer⇄Repair│  ← test_generation/verification intent
│(直出)  │  │ (Self-Correct)│
└────────┘  └──────────────┘
```
