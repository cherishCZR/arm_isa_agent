## 二、知识卡片（InstructionProfile）能力矩阵

### 2.1 可用于语义分类的字段

| 分类维度 | 可用字段 | 可靠度 | 说明 |
|---------|---------|--------|------|
| **指令标识** | `mnemonic` | ★★★★★ | 最可靠的语义标识 |
| **指令类** | `instr_class` | ★★★★☆ | general / fpsimd / sve / advsimd / mortlach2 |
| **别名信息** | `is_alias`, `alias_of` | ★★★★★ | 直接告诉我们别名和正典指令的关系 |
| **操作数信息** | `operands[].operand_type` | ★★★★★ | register / immediate / memory / label / shift / condition |
| | `operands[].register_class` | ★★★★☆ | W/X/B/H/S/D/Q/V/Z/P |
| | `operands[].register_width` | ★★★★☆ | 32/64/128 |
| | `gp_register_count` | ★★★★☆ | 通用寄存器操作数数量 |
| | `simd_register_count` | ★★★★☆ | SIMD/FP 寄存器数量 |
| | `sv_register_count` | ★★★★☆ | SVE 寄存器数量 |
| | `has_special_register` | ★★★☆☆ | SP/WZR/XZR 参与 |
| **编码信息** | `encodings[].assembly_template` | ★★★★★ | 完整汇编语法 |
| | `encodings[].name` | ★★★☆☆ | 编码名称含 32/64 |
| | `has_32bit_encoding` | ★★★★☆ | |
| | `has_64bit_encoding` | ★★★★☆ | |
| **立即数** | `immediate_ranges[].{operand_symbol, bit_width, min/max}` | ★★★★★ | 边界值依据 |
| **移位扩展** | `has_shift_extend` | ★★★★★ | 模板里有 shift/extend 可选段 |
| **标志位** | `affects_flags` | ★★★★☆ | 是否修改 NZCV |
| **约束** | `constrained_unpredictable`, `encoding_undefined` | ★★★★☆ | 非法操作数组合 |
| **架构** | `feature_dependencies` | ★★★★★ | 特性依赖 |
| **寄存器数量** | `gp_register_count`, `simd_register_count`, `sv_register_count` | ★★★★☆ | 决定 test vector 的输入数量 |

### 2.2 无法直接获取但可以派生的信息

| 派生项 | 派生方法 |
|--------|---------|
| **操作数 arity（几元操作）** | `gp_register_count` 中区分 dst vs src（按 symbol 位置 + encoding template 判断） |
| **是否纯目标操作** | 首个 register operand 通常为 dst |
| **是否有写回** | assembly_template 含 `<Xd|SP>` 等标记，或 constraints 含写回条件 |
| **指令是否影响内存** | operands 含 memory 类型 或 instr_class == load/store |
| **指令是否是分支** | mnemonic 匹配 B/BL/BLR/BR/RET/CBZ/CBNZ/TBZ/TBNZ 等 |
| **宽度限定** | 从 operands 的 register_class + register_width 推导 |

## 三、指令语义分类体系（Semantic Family）

### 3.1 分类策略

优先级：**mnemonic 匹配 > alias_of 匹配 > operand 模式匹配 > instr_class 猜测**

```
InstructionProfile (知识卡片)
    │
    ▼
┌────────────────────────────┐
│ 1. Mnemonic 精确匹配       │  ← 最优先，覆盖所有已知 ARM mnemonics
│    (known_mnemonic_map)    │
└────────┬───────────────────┘
    │ 未命中
    ▼
┌────────────────────────────┐
│ 2. Alias_of 解析           │  ← 别名指令：查父指令语义
│    profile.is_alias        │
│    → profile.alias_of      │
└────────┬───────────────────┘
    │ 不是别名/未命中
    ▼
┌────────────────────────────┐
│ 3. Operand 模式匹配        │  ← 从 operand_type 分布推断
│    gp_count + simd_count   │
│    + has_imm + has_mem     │
└────────┬───────────────────┘
    │ 模糊
    ▼
┌────────────────────────────┐
│ 4. Instr_class 兜底        │  ← 大类推断
│    general → arith/logic   │
│    fpsimd/advsimd → simd   │
│    sve → vector            │
└────────────────────────────┘
```

### 3.2 语义族定义

#### 族 1：ARITH_ADD — 加法族
```
语义: dst = (src1 + src2) [+ imm] [± extend/shift]
成员: ADD, ADDS, ADR, ADRP, CMN
test_vector_fn: (a, b) → a + b
32-bit 溢出: 结果 mod 2^32
64-bit 溢出: 结果 mod 2^64
标志影响: ADDS/CMN 设置 NZCV
```

#### 族 2：ARITH_SUB — 减法族
```
语义: dst = src1 - src2 [± imm] [± extend]
成员: SUB, SUBS, CMP, NEG, NEGS
test_vector_fn: (a, b) → a - b
32-bit 溢出: 结果 mod 2^32 (按补码解释)
64-bit 溢出: 结果 mod 2^64
标志影响: SUBS/CMP/NEGS 设置 NZCV
```

#### 族 3：ARITH_MUL — 乘法族
```
语义: dst = src1 × src2 [+ accumulator]
成员: MUL, MADD, MSUB, SMULL, UMULL, SMADDL, UMADDL, SMSUBL, UMSUBL
test_vector_fn: (a, b) → a * b [± acc]
32-bit: 低32位截断（MUL）；完整64位（SMULL/UMULL）
```

#### 族 4：ARITH_DIV — 除法族
```
语义: dst = src1 ÷ src2 (无符号/有符号)
成员: SDIV, UDIV
test_vector_fn: (a, b) → a / b (整数除法)
特殊: b=0 → 结果为 0（ARM 规范）
```

#### 族 5：LOGICAL_AND — 按位与族
```
语义: dst = src1 & src2 [~ (取反)]
成员: AND, ANDS, BIC, BICS, TST
test_vector_fn: (a, b) → a & b (BIC: a & ~b)
```

#### 族 6：LOGICAL_OR — 按位或族
```
语义: dst = src1 | src2 [~ (取反)]
成员: ORR, ORN, MOV (alias of ORR with XZR)
test_vector_fn: (a, b) → a | b (ORN: a | ~b, MOV: 0 | b = b)
```

#### 族 7：LOGICAL_XOR — 按位异或族
```
语义: dst = src1 ^ src2 [~ (取反)]
成员: EOR, EON, MVN (alias of ORN with XZR)
test_vector_fn: (a, b) → a ^ b (EON: a ^ ~b; MVN: 0 | ~b = ~b)
```

#### 族 8：SHIFT — 移位族
```
语义: dst = src1 << src2  /  src1 >> src2  /  rotate(src1, src2)
成员: LSL, LSR, ASR, ROR, LSLV, LSRV, ASRV, RORV
test_vector_fn: (a, b) → a << b / a >> b / ROR(a, b%width)
```

#### 族 9：MOVE — 数据搬运族
```
语义: dst = 构造后的立即数/寄存器值
成员: MOVZ, MOVN, MOVK, MOV (wide imm alias), FMOV, MOV (register alias)
特点: 操作数 arity 可变（1源1目标 或 立即数构造）
test_vector_fn: (a) → a (寄存器搬运) / imm → explicit_pattern (立即数构造)
```

#### 族 10：COMPARE — 纯标志比较族
```
语义: 仅设置 NZCV，不修改目标寄存器
成员: CMP, CMN, TST
特点: dst 隐式 XZR，结果被丢弃
```

#### 族 11：BITFIELD — 位域操作族
```
语义: 位域插入/提取/移动
成员: BFM, UBFM, SBFM, BFI, BFXIL, UBFX, SBFX, EXTR
特点: 参数化位宽和位置
```

#### 族 12：REVERSE — 位/字节反转族
```
语义: dst = 反转后的 src
成员: REV, REV16, REV32, RBIT, CLZ, CLS
```

#### 族 13：SELECT — 条件选择族
```
语义: dst = condition ? src1 : src2
成员: CSEL, CSINC, CSINV, CSNEG, CSET, CSETM, CINC, CINV, CNEG
```

#### 族 14：LOAD — 加载族
```
语义: dst = Mem[addr + offset]
成员: LDR, LDUR, LDP, LDNP, LDXR, LDAXR, LDTR, LDAR, LDLAR, etc.
```

#### 族 15：STORE — 存储族
```
语义: Mem[addr + offset] = src
成员: STR, STUR, STP, STNP, STXR, STLXR, etc.
```

#### 族 16：BRANCH — 分支族
```
语义: PC = target
成员: B, BL, BLR, BR, RET, CBZ, CBNZ, TBZ, TBNZ
```

#### 族 17：SIMD_VECTOR — 向量/SIMD 族
```
语义: 向量操作（每个 lane 独立运算）
成员: 所有 instr_class=fpsimd/advsimd/sve 的指令
```

## 四、各语义族的 Test Vector 生成规则

### 4.1 核心原则

1. **通用模板**：每个族定义一组固定的 test vectors + 对应的 reference 计算函数
2. **宽度感知**：32-bit 模式全部 masked `& 0xFFFFFFFF`，64-bit 全宽
3. **标志感知**：affects_flags 的指令额外生成 NZCV 预期值
4. **立即数感知**：有 imm operand 时调整输入策略（单寄存器 + 多立即数）

## 五、汇编模板实例化规则（Template Instantiation）

### 5.1 问题分析

当前 `_instantiate_template()` 的 `str.replace()` 方法无法处理以下 ARM 模板语法：

```
模板: ADD  <Wd|WSP>, <Wn|WSP>, <Wm>{, <extend> {#<amount>}}
      ↑              ↑          ↑      ↑ 可选段 ...

当前 replace 只替换了 <Wd|WSP> → X0, <Wn|WSP> → X1, <Wm> → X2
但 <extend>、<amount> 没有出现在 profile.operands 中（或出现了但 symbol
不匹配 operands 表中的 symbol），因此未被替换。
```

### 5.2 新的模板实例化策略

采用**两阶段处理**：

**阶段 1：可选段剪枝**
```
输入: "ADD  <Wd|WSP>, <Wn|WSP>, <Wm>{, <extend> {#<amount>}}"
      → 识别所有 { ... } 括号段
      → 对于 {, <extend> {#<amount>}}，如果 instruction 不需要扩展
        （由生成器上下文决定），整段删除
      → 输出: "ADD  <Wd|WSP>, <Wn|WSP>, <Wm>"
```

剪切规则：
- `{, 可选段}` → 如果本次测试不测这个可选段，删除 `{, 可选段}`
- 对于 `{<extend> {#<amount>}}` → 嵌套展开或删除

**阶段 2：占位符替换**
```
输入: "ADD  <Wd|WSP>, <Wn|WSP>, <Wm>"
      → 对每个 <...> 占位符：
        1. 检查 profile.operands 中是否有 matching symbol
        2. 如果有，按 operand_type 映射到具体值
        3. 如果没有（模板中有但 operands 表中缺），用模板解析得到 symbol base 名
        4. 如果是 register 类型 → 分配下一个可用寄存器
        5. 如果是 immediate 类型 → 用合理的值（如 #0）
        6. 如果是 shift/extend 类型 → 用简单的值（如 LSL #0）
```

**阶段 2 详细替换规则**：

| 占位符类型 | 替换策略 |
|-----------|---------|
| `<[WX]d>` | 第一个可用寄存器，排除特殊寄存器 |
| `<[WX]n>` | 第二个可用寄存器 |
| `<[WX]m>` | 第三个可用寄存器 |
| `<[WX]s>` | 第四个可用寄存器 |
| `<[WX]t>` | 第五个可用寄存器 |
| `<Wd\|WSP>` | 如果测试 SP→选择 SP，否则选寄存器 |
| `<imm>` / `#<imm>` | 选一个合理值：`#0` 是最安全的默认值 |
| `<label>` | 如 `label_1f` 或 `target` |
| `<extend>` | `LSL` (UXTW 对 32-bit, UXTX 对 64-bit) |
| `<amount>` | `#0` |
| `<shift>` | 如有值表→用第一个值（如 `LSL #0`），否则 `#0` |
| `<p>`（谓词） | 如 `P0` |
| `<T>`（数组类型） | 如 `2S`，依赖上下文 |
| `<R>`（宽度说明符） | `W` 或 `X` |
| `<Vd>.<T>` | 如 `V0.2S` |
| `<condition>` | `EQ` 或 `AL` |
| `{可选段}` | 如果包含 shift/extend 的选项→默认**不展开（删除该可选段）** |

### 5.3 关键：operands 表不一定包含所有模板占位符

从 XML 解析流程看：
- `<asmtemplate>` 中的 `<a>` 元素 → `operand_symbols`（如 `<Wm>`, `<extend>`, `<amount>`）
- `<explanations>` 中的 `<explanation>` → `operands` 表

**这两个列表不一定一一对应**。有些占位符只在模板中出现但不在 explanations 中。

**解决方法**：模板解析不仅依赖 operands 表，还要从 `assembly_template` 字符串中直接提取所有 `<...>` 占位符，然后分别处理：
1. 能在 operands 表找到的 → 按 operand_type 替换
2. 找不到的 → 按占位符名称模式（register/immediate/label）做最佳猜测替换

```python
def instantiate_template_robust(template: str, profile, reg_prefix: str) -> str:
    """Robust template instantiation. Handles ALL ARM template syntax."""
    import re

    # Step 0: Remove optional sections for non-shift-extend tests
    #         (unless the test is specifically about shift/extend)
    inst = re.sub(r'\{,\s*<[^>]+>\s*\{#<[^>]+>\}\}', '', template)  # {, <extend> {#<amount>}}
    inst = re.sub(r'\{,\s*<[^>]+>\}', '', inst)   # {, <shift>}

    # Step 1: Extract all placeholders from template
    placeholders = re.findall(r'<([^>]+)>', inst)
    
    # Step 2: Build operand lookup: symbol → OperandSummary
    op_map = {op.symbol.strip('<>'): op for op in profile.operands}
    
    # Step 3: Allocate registers sequentially
    reg_idx = [0]  # use list for closure
    def next_reg(prefer_64=True, exclude_special=True):
        r = _select_register(reg_idx[0], prefer_64, exclude_special)
        reg_idx[0] += 1
        return r
    
    # Step 4: Replace each placeholder
    for ph in placeholders:
        # Handle | notation: first choice is default
        # <Wd|WSP> → ['Wd', 'WSP']
        choices = ph.split('|')
        base = choices[0]
        
        # Check operands table first
        matched_op = None
        for choice in choices:
            if choice in op_map:
                matched_op = op_map[choice]
                break
        if base in op_map:
            matched_op = op_map[base]
        if not matched_op:
            # Fuzzy match: check if any operand symbol contains this
            for sym, op in op_map.items():
                if choice in sym or sym in choice:
                    matched_op = op
                    break
        
        if matched_op:
            if matched_op.operand_type == "register":
                val = next_reg(reg_prefix == 'X', exclude_special=True)
            elif matched_op.operand_type == "immediate":
                val = "#0"
            elif matched_op.operand_type in ("shift", "option"):
                val = "LSL #0"
            elif matched_op.operand_type == "label":
                val = "1f"
            elif matched_op.operand_type == "condition":
                val = "al"
            else:
                val = "#0"
        else:
            # Guess by pattern
            ph_lower = ph.lower()
            if any(kw in ph_lower for kw in ['w', 'x', 'd', 'n', 'm', 's', 't']):
                # Register-like
                if ph_lower.startswith('w'):
                    val = next_reg(prefer_64=False, exclude_special=True)
                elif ph_lower.startswith('x'):
                    val = next_reg(prefer_64=True, exclude_special=True)
                elif any(kw in ph_lower for kw in ['v', 'q', 'd.', 's.', 'h.', 'b.']):
                    val = 'V0.2S'  # placeholder for SIMD
                else:
                    val = next_reg(prefer_64=(reg_prefix == 'X'), exclude_special=True)
            elif 'imm' in ph_lower or 'amount' in ph_lower:
                val = "#0"
            elif 'label' in ph_lower:
                val = "1f"
            elif 'extend' in ph_lower or 'shift' in ph_lower:
                val = "LSL #0"
            elif 'p' == ph_lower[:1].lower():
                val = "P0"  # SVE predicate
            else:
                val = "#0"  # fallback
        
        # Replace all forms of this placeholder
        # Handle <Wd|WSP> style: replace entire placeholder
        full_ph = f"<{ph}>"
        # Also handle #<imm> form
        if full_ph in inst:
            inst = inst.replace(full_ph, val)
        elif f"#<{ph}>" in inst:
            inst = inst.replace(f"#<{ph}>", val)
    
    # Step 5: Clean up template artifacts (single remaining <, >)
    inst = re.sub(r'\s*,?\s*<[^>]*>', '', inst)
    
    return inst.strip()
```

```c
// 32-bit 变体（错误）:
// 用 .replace('r"', 'w"') 把约束从 "r" 改为 "w" (SIMD reg)
// ADD/SUB/MOV 是通用寄存器指令，"w" 约束完全错误
```

### 6.2 新规则

**规则 1：操作数约束选择**

| 指令语义族 | 输出约束 | 输入约束 | 说明 |
|-----------|---------|---------|------|
| ARITH_ADD/SUB/MUL/DIV | `"=&r"` | `"r"` | 通用寄存器 |
| LOGICAL_AND/OR/XOR | `"=&r"` | `"r"` | 通用寄存器 |
| SHIFT | `"=&r"` | `"r"` | 通用寄存器 |
| MOVE | `"=&r"` | `"r"` / `"i"`（立即数） | 通用寄存器 |
| fpsimd/advsimd/sve | `"=&w"` | `"w"` | FP/SIMD 寄存器 |
| LOAD/STORE | `"=r"` / `"=&r"` | `"r"`（地址） + `"m"`（内存） | |

**规则 2：约束字符串不可简单 replace**
```
错误：.replace('r"', 'w"')  ← 可能误替换字符串值中的 'r"'
正确：按 operand 类型逐一构建约束列表
```

**规则 3：输入/输出操作数的正确对应**
```
对于 "sub X0, X1, X2":
  output: [out0] "=&r" (result)    ← X0
  inputs: [in1]  "r"   (a),        ← X1
          [in2]  "r"   (b)         ← X2
  clobber: "cc" (if SUBS)
  
模板: "%[out0], %[in1], %[in2]"  ← 不使用 %0 %1 %2，始终用命名操作数
```

**规则 4：寄存器计数决定操作数数量**
```python
def build_inline_asm_operands(profile, semantic_family):
    n_gp = profile.gp_register_count
    # 第一个 GP 寄存器通常是输出
    out_ops = ['[out0] "=&r" (result)' if n_gp >= 1 else '']
    in_ops = []
    for i in range(1, n_gp):
        in_ops.append(f'[in{i}] "r" (in{i})')
    
    # 立即数操作数
    for imm in profile.immediate_ranges:
        in_ops.append(f'[imm_{imm.operand_symbol}] "I" (imm_val)')
    
    return out_ops, in_ops
```

// 对于 "sub X0, X1, X2":
static inline uint64_t run_instruction(uint64_t a, uint64_t b) {
    uint64_t result;
    asm volatile(
        "sub %[out], %[in1], %[in2]\n"
        : [out] "=&r" (result)         // X0 → result
        : [in1] "r" (a), [in2] "r" (b) // X1 → a, X2 → b
        : // no clobber for SUB (non-flags)
    );
    return result;
}

// 对于 "adds X0, X1, X2" (affects_flags):
static inline uint64_t run_instruction(uint64_t a, uint64_t b) {
    uint64_t result;
    asm volatile(
        "adds %[out], %[in1], %[in2]\n"
        : [out] "=&r" (result)
        : [in1] "r" (a), [in2] "r" (b)
        : "cc"                        // flags clobbered
    );
    return result;
}

```python
def generate_boundary_test(profile, semantic_family, encoding_width):
    """
    根据语义族和位宽生成正确的 boundary test。
    """
    mnemonic = profile.mnemonic.lower()
    is_64bit = encoding_width == 64
    mask = 0xFFFFFFFFFFFFFFFF if is_64bit else 0xFFFFFFFF
    reg_prefix = "X" if is_64bit else "W"
    
    # 根据语义族选择 reference_fn
    ref = _get_reference_fn(semantic_family, is_64bit)
    
    tests = []
    
    # 1. Zero input test
    tests.append({
        "test_name": f"test_{mnemonic}_zero",
        "input":     {f"{reg_prefix}1": "0x0", f"{reg_prefix}2": "0x0"} if n_gp >= 3 else ...,
        "expected":  {f"{reg_prefix}0": hex(ref(0, 0) & mask)},
        "category":  "zero",
        "behavior":  f"Zero inputs: {semantic_family.name}(0,0) = {ref(0,0)}",
    })
    
    # 2. Width boundary: 32-bit overflow
    if not is_64bit:
        tests.append({
            "test_name": f"test_{mnemonic}_w_reg_32bit",
            "input":     {"W1": "0xFFFFFFFF", "W2": "0x1"},
            "expected":  {"W0": hex(ref(0xFFFFFFFF, 1) & 0xFFFFFFFF)},
            "category":  "width_boundary",
            "behavior":  f"32-bit {semantic_family.name}: 0xFFFFFFFF op 0x1 = {hex(ref(0xFFFFFFFF, 1) & 0xFFFFFFFF)}",
        })
    
    # 3. Width boundary: 64-bit
    if is_64bit:
        tests.append({
            "test_name": f"test_{mnemonic}_x_reg_64bit",
            "input":     {"X1": "0xFFFFFFFF", "X2": "0x1"},
            "expected":  {"X0": hex(ref(0xFFFFFFFF, 1))},
            "category":  "width_boundary",
            "behavior":  f"64-bit {semantic_family.name}: 0xFFFFFFFF op 0x1 = {hex(ref(0xFFFFFFFF, 1))}",
        })
    
    # 4. Src/dst overlap
    test_val_a = 0x5
    test_val_b = 0x3
    tests.append({
        "test_name": f"test_{mnemonic}_src_dst_overlap",
        "input":     {f"{reg_prefix}0": hex(test_val_a), f"{reg_prefix}1": hex(test_val_b)},
        "expected":  {f"{reg_prefix}0": hex(ref(test_val_a, test_val_b) & mask)},
        "category":  "register_edge",
        "behavior":  f"Source/dest overlap: {semantic_family.name}({test_val_a}, {test_val_b}) = {ref(test_val_a, test_val_b)}",
    })
    
    # 5. Zero register destination (if has_special_register)
    if profile.has_special_register:
        tests.append({
            "test_name": f"test_{mnemonic}_zero_reg_dst",
            "input":     {f"{reg_prefix}1": "0x10", f"{reg_prefix}2": "0x20"},
            "expected":  {},  # XZR 丢弃结果
            "category":  "zero",
            "behavior":  "Result discarded to XZR/WZR",
        })
    
    return tests


def _get_reference_fn(family, is_64bit):
    """Return a Python function that computes expected result."""
    mask = 0xFFFFFFFFFFFFFFFF if is_64bit else 0xFFFFFFFF
    
    if family == SemanticFamily.ARITH_ADD:
        return lambda a, b: (a + b) & mask
    elif family == SemanticFamily.ARITH_SUB:
        return lambda a, b: (a - b) & mask
    elif family == SemanticFamily.ARITH_MUL:
        return lambda a, b: (a * b) & mask
    elif family == SemanticFamily.ARITH_DIV:
        return lambda a, b: (a // b) & mask if b != 0 else 0
    elif family == SemanticFamily.LOGICAL_AND:
        return lambda a, b: (a & b) & mask
    elif family == SemanticFamily.LOGICAL_OR:
        return lambda a, b: (a | b) & mask
    elif family == SemanticFamily.LOGICAL_XOR:
        return lambda a, b: (a ^ b) & mask
    elif family == SemanticFamily.SHIFT:
        return lambda a, b: (a << (b & 63)) & mask
    elif family == SemanticFamily.MOVE:
        return lambda a, b: a & mask  # b 被忽略
    else:
        return lambda a, b: 0
```

    // Expected: X0 = 0xFFFFFFFFFFFFFFF0   ← 正确！0x10 - 0x20 = -0x10 (补码)
    ret

test_sub_zero_reg:

### 10.4 C++ Verification（新规则）

```cpp
// Reference model: X0 = X1 - X2
static inline uint64_t reference_model(uint64_t a, uint64_t b) {
    return a - b;                       // ← 正确实现，不再是 return 0
}

static inline uint64_t run_instruction(uint64_t a, uint64_t b) {
    uint64_t result;
    asm volatile(
        "sub %[out], %[in1], %[in2]\n"
        : [out] "=&r" (result)
        : [in1] "r" (a), [in2] "r" (b)
        : /* no cc clobber for SUB (non-flags variant) */
    );
    return result;
}

// Test 1: a=0, b=0
// Expected: result=0 (0 - 0 = 0)      ← 正确！
// Test 2: a=1, b=1
// Expected: result=0 (1 - 1 = 0)      ← 正确！不再是 2
// Test 3: a=0xFFFFFFFF, b=1
// Expected: result=0xFFFFFFFE          ← 正确！不再是 0x100000000
```

## 十一、总结

### 11.1 核心改进

| 维度 | 当前 | 新设计 |
|------|------|--------|
| 指令语义 | 全部当作 ADD | 精确匹配已知 mnemonic + operand 模式推断 |
| Test Vectors | 硬编码 `0x10+0x20=0x30` | 按语义族 Python reference_fn 实时计算 |
| 模板实例化 | 简单 str.replace | 两阶段：可选段剪枝 + robust 占位符替换 |
| reference_model | `return 0 // TODO` | 按语义族生成正确的 C 表达式 |
| 内联汇编约束 | 固定 `"r"(a), "r"(b)` | 按 operands 计数动态生成，命名操作数 |
| alobber | 固定 `"cc"` | 仅 affects_flags 时加 `"cc"` |
| 32-bit 变体 | `.replace('r"', 'w"')` | 始终用 `"r"` 对 GP 指令，只在 SIMD 族用 `"w"` |

### 11.2 代码量估算

- 新增 `semantics.py`：约 400-500 行
- 修改 `generators.py`：约 200 行变更（替换 rule-based 方法）
- 修改 `models.py`：约 80 行变更（to_cpp_program 和 to_assembly_file）
- 总改动：约 700-800 行

### 11.3 不依赖 LLM

新设计完全基于规则（deterministic），不依赖 LLM 调用。这与当前批量验证默认 `use_llm=False` 的设计保持一致。
